2025-02-14 - [Numba Optimization: `swing_noise` in `batch_compute_states`]
Learning: When computing running drawdown and drawup over rolling segments (like `swing_noise_ticks`), the combination of pure Python `for` loops, multiple array slice allocations per loop (e.g. `_seg_hi = highs[_ni - _noise_window:_ni + 1]`), and `np.maximum.accumulate` incurs significant overhead (`~0.23s` per 20k rows). Rewriting this operation into a single `@njit(parallel=True, cache=True)` Numba block that calculates the drawdown and drawup scalar values directly drops the computation to `~0.0006s`, a ~300x speedup, bypassing PyObject overhead and intermediate array generation.
Action: Whenever `np.maximum.accumulate` or `sliding_window_view` is used in a per-bar or sequential rolling loop to track peaks/drawdowns, refactor into an `@njit(cache=True, parallel=True)` numba function using `numba.prange` to handle independent windows.
