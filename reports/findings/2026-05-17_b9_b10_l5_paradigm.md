## Detailed results

### B9 v1 (binary cut, target `exit_pnl < -$50`)

Walk-forward across 4 folds, 5 K horizons, 3 recall budgets = 60 ops:

```
K     recall    AUC    mean Δ/day   avg per-fold CI       sig
 5    0.50    0.861    -$81         [-$182, -$2]          0/4
30    0.50    0.904    -$51         [-$109, -$2]          0/4
60    0.50    0.920    -$10         [-$44, +$20]          0/4
120   0.30    0.930    -$2          [-$18, +$13]          1/4
```

**Diagnosis**: model has high AUC (0.86-0.93) — it CAN rank bad trades correctly. But cutting on its predictions loses money because mid-MAE trades that look bad often RECOVER. Type 1 cost > Type 2 savings.

This replicates the morning's trail-tightening failure with a different model. **R-trigger is structurally optimal under binary cut/hold.**

### B9 v2 (aligned target `cut_savings > $20`)

Same walk-forward methodology, reframed target:
- y=1 iff (pnl_so_far - exit_pnl) > $20, i.e., "cutting at K saves at least $20"

```
K     recall    AUC    mean Δ/day   avg per-fold CI       sig
60    0.50    0.638    +$8          [-$80, +$89]          1/4
120   0.50    0.598    +$29         [-$36, +$88]          1/4
120   0.70    0.598    +$32         [-$44, +$104]         0/4
```

AUC drops sharply (0.60-0.64) because the target is harder (binary "should I cut" requires the model to predict the SIGN and MAGNITUDE of remaining $). No operating point hits ≥2/4 folds significant.

**Verdict**: even with the operationally aligned target, binary cut is structurally negative on average.

### B9 (continuous remaining-amplitude regressor)

Per-K HistGradientBoosting regressors on V2 + trajectory features (190 features). Target: `remaining_pnl_usd = exit_pnl_usd - pnl_usd_so_far`. Sizing rule maps prediction magnitude to {0, 0.5, 1.0, 1.5}.

**IS walk-forward (4 expanding folds, 275 days)**:

```

 5    0.217     $42.33    +$48         [+$24, +$74]          3/4 ★
10    0.172     $41.69    +$13         [-$16, +$42]          2/4
30    0.114     $38.40    -$2          [-$15, +$13]          0/4
60    0.126     $36.40    +$15         [+$1, +$31]           2/4
120   0.088     $34.15    +$9          [-$12, +$29]          2/4
```

Pearson is modest (0.09-0.22) but **the continuous sizing action survives walk-forward where binary cut fails**. K=5 strongest: 3/4 folds significant.

### Anti-doom-cascade slippage stress test

Assumes each ACTION (size_factor != 1.0) incurs additional slippage $S beyond the $6/leg friction baseline:

```
K       S=$0      S=$2      S=$5      S=$10
 5     +$67     +$61     +$53     +$38
10     +$33     +$24     +$12      -$8
30     +$16     +$13      +$9      +$2
60      +$1      -$0      -$2      -$5
120     +$4      -$8     -$25     -$55
```

K=5 maintains +$38/day even under $10/action slippage. K=120 is most slippage-sensitive (many half-actions amplify the cost).

## Why B9 works where B9 doesn't

1. **Variable action**: B9 outputs {0, 0.5, 1.0, 1.5}. Only HIGH-CONFIDENCE negative predictions get size=0 (cut). Most legs get 1.0 (no action, no Type 1 cost).
2. **Magnitude-aware**: B9 cut at any P > threshold; B9 only cuts when prediction is strongly negative. Marginal predictions → hold.
3. **Pyramids capture upside**: B9 scales UP on strong positive predictions, capturing more gain from clear winners. B9 had no equivalent.
4. **Smoother loss surface**: B9 trained with regression loss; gradient signal is informative across the prediction range. B9's binary cross-entropy loss treats marginal cases the same as confident cases.

## Failure modes to monitor

1. **K=5 is fast** — predictions at 25 seconds carry less info than predictions at 60-120 seconds. But K=5 is also the highest CI-positive result. Why? Hypothesis: at K=5, MAE/MFE haven't diverged much from entry, so V2 features dominate. By K=60+, trajectory features dominate but are noisier. Worth investigating but DO NOT retune.

2. **Pearson 0.217 is low**. The model is weak at point prediction. The win comes from the RANK ORDER + threshold action, not from precise magnitude. This means small calibration drifts can flip individual decisions. Need monitoring.

3. **88 pyramids on OOS**. These add upside but also concentrate risk. Live deployment should cap pyramids at 1.25x initially (not 1.5x) until live behavior is observed.

4. **Worst day -$86**. Below baseline best day. Within expected range but should be monitored. If live worst-day exceeds -$200, investigate.

## Compounding with B7

B9 is a SECOND sizing layer on top of B7 (entry-time leg sizer). The two are complementary:
- B7 sets initial size based on predicted leg amplitude (1.0x baseline, scaled by gbm_ev rule)
- B9 adjusts at T+25s based on actual trajectory

If B7 oversizes and the trade is going badly, B9 can REDUCE. If B7 undersizes and the trade is going strongly, B9 can PYRAMID. They multiply, not add.

Empirical baseline:
- Flat sizing: $475/day
- B7 GBM sizing: $927/day (= +$452/day = +95%)
- B7 + B9 K=5: expected ~$994/day if B9's +$67 stacks linearly on B7's $927 (CI-bounded ~ +$1,000-1,030/day)

But linear stacking is an assumption — B7 already uses some of the information B9 uses (V2 features). The honest estimate: B7 + B9 ≥ $927 with B9 adding $30-67/day incrementally. Real number requires a paired backtest.
