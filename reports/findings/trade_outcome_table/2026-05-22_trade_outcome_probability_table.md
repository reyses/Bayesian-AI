# Trade Outcome Probability Table

Diagnostic probability table over every **hardened zigzag-leg trade** (the legs the engine actually traded, R-trigger exits). Three per-trade quantities: **entry-to-close** P&L, **MAE** (max adverse excursion), **MFE** (max favorable excursion). Pure descriptive statistics -- no model fit, no leakage. Generated by `tools/trade_outcome_probability_table.py`.

## Method

- entry-to-close = CSV `pnl_pts`/`pnl_usd` (authoritative, net of $6/leg friction; MNQ = $2/point).
- MAE/MFE reconstructed from the full 5s intra-trade path (entry->exit).
- **Key identity**: "the trade ever reached open-PnL level x" == `MFE >= x` (running maximum). So the continuation/recovery tables are MFE/MAE threshold conditionals -- no per-bar path needed.
- Two axes: **R-multiples** (excursion / `r_price`; regime-invariant, cross-comparable) and **ticks** (raw, 0.25-pt). Both forms: survival (CDF) and histogram (PDF).
- Every conditional cell: `n` + 95% bootstrap CI (4000 resamples). Cells with n < 30 are flagged ` !`.

## Caveats -- read before using

1. **`close ~= MFE - R`**: a zigzag leg exits ~`r_price` below its favorable peak by construction, so every winner gives back ~1R. The continuation table shows `close` and `MFE` side-by-side so this stays visible -- "continue beyond +$200" is largely the MFE distribution shifted down by ~1R.
2. The **recovery table describes the existing R-trigger exit** -- it says how often that rule still wins from a drawdown; it does NOT prove holding is optimal. `tools/drawdown_stop_analysis.py` already settled "cut at -$100 loses, 76% of -$100 legs recover".
3. **IS and OOS are reported separately, never pooled** (OOS legs run larger -- known regime shift). Trust a cell only where IS and OOS agree.

## Read

- **Continuation**: scan table B `P(close>=$200)` down the rows -- it shows how the odds of a big winner change once the trade has already shown `+x`. Compare `P(close>=x) holds` against `P(MFE>=x+1R) extends`: the gap is the structural ~1R giveback.
- **Recovery**: scan table C `P(close>0) RECOVERY` -- a drawdown of `-d` is survivable only while that probability (and its CI) stays well above 50%. Where it crosses below, the existing exit rarely rescues the trade -- but see caveat 2 before acting on it.
- A cell is trustworthy only where the IS and OOS rows agree and the CI is tight; treat ` !`-flagged thin cells as direction-only.
