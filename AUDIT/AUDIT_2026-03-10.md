## 3. STRUCTURE AND CLEANLINESS AUDIT

The project generally follows an excellent structural separation between `core/`, `live/`, `training/`, `visualization/`, `tools/`, and `scripts/`. However, some deviations were found during the file hierarchy audit:

### Identified Misplacements:
1. **`scripts/hurst_validation.py`**
   - Currently resides in the `scripts/` root directory.
   - **Recommendation:** Move to `scripts/debug/` as it functions primarily as a diagnostic validation tool for ADX vs. Hurst logic. A `scripts/debug/` directory was created to hold it.

2. **Research Scripts Organization**
   - The separation between `scripts/research/` (e.g., `analysis_v_trajectory_knn.py`) and `tools/research/` (`plots.py`, `imr.py`) works but requires strict adherence to avoid blurring lines. Standalone test/research scripts live in `scripts/research/` while utilities reside in `tools/research/`. This is acceptable but must be strictly enforced.

## 4. DEBUG AND CLI TOOLING AUDIT

A scan of all auxiliary scripts (specifically `scripts/hurst_validation.py` and those under `scripts/research/`) was conducted to evaluate logging and argument parsing.

### Argparse Usage
- **Result:** Excellent. All standalone executable scripts utilize `argparse` to handle arguments (e.g., `--data`, `--base_tf`, `--k`) appropriately.

### Logging Usage
- **Result:** Deficient. All execution outputs in `scripts/hurst_validation.py` and the `scripts/research/analysis_*.py` files rely entirely on standard `print()` statements.
- **Rule Violation:** Project rules state: _"Debug scripts in `scripts/debug/` and `training/orchestrator.py` must use `logging` and `argparse` instead of `print()` for production/operational output."_
- **Impact:** Output cannot be reliably routed, timestamped, or filtered by severity when run automatically or scaled up.
