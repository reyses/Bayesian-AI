2025-02-27 - [Optimize Swing Noise Computation with Numba `prange`]
Learning: Using `numpy.maximum.accumulate` or `numpy.minimum.accumulate` within a loop that continually takes small slices of arrays incurs high PyObject overhead. For rolling running drawdown/drawup calculations (e.g. swing noise), parallel direct scalar computation using Numba `@njit(parallel=True)` with `prange` yields huge performance gains (>160-370x) because it eliminates intermediate array allocations per bar entirely. The critical numerical insight: initialize the running memory difference to the first element (e.g., `run_hi - run_lo`) rather than `0.0` to preserve mathematical parity with NumPy.
Action: Whenever tracking intra-wave metrics like max drawdown inside a rolling window, avoid sliced `accumulate` arrays; instead write custom, explicitly parallelized Numba kernels that manually track running extrema.
