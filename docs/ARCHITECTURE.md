## System Overview
Bayesian-AI is a high-frequency trading system that models market price action as a "Price Particle" moving through a "Quantum Field" defined by gravitational attractors (Fair Value, Support, Resistance). It employs a "Load -> Transform -> Analyze -> Visualize" pipeline to ingest high-resolution data (15s bars), identify recurring fractal patterns, and execute trades based on probabilistic outcomes.

The core engine uses a "Bayesian Brain" with dynamic binning and a "Fractal Clustering Engine" (Recursive K-Means) to learn probability distributions of market states. A "Timeframe Belief Network" aggregates conviction across 8 timeframe levels (1h to 15s) to drive entry decisions. Trade management is handled by a "Wave Rider" that dynamically adjusts trailing stops based on spectral analysis (Fourier/Laplace gates) and post-trade regret markers.

## Key Constants & Parameters
- `min_bins`/`max_bins` (DynamicBinner): 5 / 30 — Binning resolution for state discretization.
- `HURST_EXPONENT` threshold: 0.5 — Below this is considered "Chop/Noise" (execution halted).
- `tunnel_probability`: 0.80 — Minimum probability required to enter a tunnel trade.
- `MIN_PATTERNS_FOR_SPLIT`: 30 — Minimum patterns required to split a cluster further.
- `R2_STOP_THRESHOLD`: 0.15 — Stop splitting if regression R² (predictive power) is sufficiently high.
- `MAX_FISSION_CLUSTERS`: 6 — Maximum number of sub-clusters created per split.
- `INITIAL_CLUSTER_DIVISOR`: 100 — Scaling factor for initial clusters.
- `TIMEFRAME_MAP`: {0: '5s', ..., 5: '1h'} — Mapping of integer levels to timeframe strings.
