# 2026-05-01 — Multi-TF v2 standalone runs + composite directional system

## TL;DR

Ran the full v2 standalone_research pipeline at 6 base TFs (1m, 5m, 15m, 1h, 4h, 1D) against `DATA/ATLAS/FEATURES_5s_v2/`. Built two composite tools (`v2_composite_directional.py` for OLS-mean, `v2_composite_quantile.py` for distributional Q_0.25/Q_0.75 voting, `v2_composite_l_aggregator.py` for direct cross-TF aggregation of standalone Analysis L outputs).

**Key finding: L's signed-MFE OLS works on own-cadence (62–71% accuracy, +2–6% lift) but naive cross-TF voting at a common 5m cadence drops to 52–53% per voter** — horizon mismatch (each TF predicts its own future window: 1m→1h ahead, 1h→8h ahead). Composite combinators salvage some lift: unanimous 5-TF agreement → 56.5% on 33% of data (+6.2% lift); magnitude-weighted |w|>5 → 59.8% on 13% (+9.6%).

**Operational read: trade per-TF L gates, not the naive composite.** A real composite system needs all TFs refit against a single common-horizon target.

**Sweet spots:**
- 1m's |pred|>20 tier: 73.1% accuracy on 20% of bars (+14.8% lift). Sharpest tail.

## Composite directional system attempts

### Approach 1 — `v2_composite_directional.py` (OLS-mean refit per TF, then vote)
Per-bar target = `max(MFE_long, MFE_short) * sign(winner_leg)`. Each bar can flip sign independently. Result: R² ≈ −0.06 to −0.10, accuracy ≈ 49.5%. **Bad target choice** — different from standalone L's regime-segmented signed MFE. Killed.

### Approach 2 — `v2_composite_l_aggregator.py` (read standalone L's CSVs, vote)
Reads each TF's `analysis_l_predictions.csv`, projects per-TF predictions onto common 5m cadence via searchsorted, applies four combinators:

**Per-TF accuracy at 5m common cadence** (vs 50.2% baseline):
| TF | Acc | Lift |
|---|---:|---:|
| 1m | 52.4% | +2.2% |
| 5m | 52.6% | +2.3% |
| 15m | 52.5% | +2.3% |
| 1h | 53.4% | +3.2% |
| 4h | 52.3% | +2.0% |

The drop from own-cadence (62–71%) to common-cadence (52–53%) is the **horizon mismatch tax**.

**Composite combinators at 5m cadence:**
| Combinator | N | Acc | Lift | %data |
|---|---:|---:|---:|---:|
| majority ≥1 | 73,764 | 53.1% | +2.9% | 100% |
| majority ≥4 | 50,146 | 54.3% | +4.0% | 68% |
| majority ≥5 (unanimous) | 24,561 | 56.5% | +6.2% | 33% |
| strict-all (no dissent) | 24,561 | 56.5% | +6.2% | 33% |
| weighted \|w\|>3 | 24,277 | 56.8% | +6.6% | 33% |
| weighted \|w\|>5 | 9,626 | **59.8%** | **+9.6%** | 13% |
| confgated \|p\|>10 majority | 66,895 | 53.5% | +3.3% | 91% |

The **best composite** (magnitude-weighted, |w|>5) hits 59.8% on 13% of bars. That's a real signal but **still below any single-TF own-cadence gate**.

### Approach 3 — `v2_composite_quantile.py` (Q_0.25 > 0 / Q_0.75 < 0 distributional gating)
Coded but not run end-to-end yet. Would replace OLS-mean with conditional quantile regression (lightgbm or sklearn GBM). This is the operationalization of the "predict the distribution, not the point" framing. Per-TF quantile fits at 1m would take ~60 min on CPU; higher TFs <5 min each. Pending decision on whether to run.

## What this tells us about distributional vs point prediction

User's framing earlier: "we have the same problem classical physics has when evaluating electron position — by the time it reacts the electron is somewhere else; it might be right all along to treat price as a probability field."

Operationally validated:
- Bar-to-bar **direction sign** → R² = 0.0003 (Analysis B). Coin flip.
- **Conditional mean of signed MFE** (Analysis L) → R² = 0.06–0.11, gates to 73% at 20% coverage. The same data, different question.

The distinction is **distributional thinking**: stop asking "where will the price be next bar" and start asking "what's the conditional distribution of the trajectory; is it concentrated enough to bet". L is already a partial answer (mean of distribution); quantile regression (`v2_composite_quantile.py`) would extend to spread/skew.

Note: the QM analogy is rhetorical — distributional finance is mainstream (every option pricer, GARCH model, quantile regressor already does this). No physics imports into code per project policy.

## What's next

Two open paths:

1. **Refit all TFs against a single common-horizon target** so the composite is comparing apples to apples. The current 56.5%/+6.2% unanimous tier underperforms single-TF gates because each voter is predicting a different horizon. A composite where all 5 voters predict "next 2-hour direction at 5m bars" (or any single chosen horizon) would be the honest test.

2. **Run the quantile composite** (`v2_composite_quantile.py`) — replaces L's mean prediction with Q_0.25/Q_0.75-based "is the distribution unambiguously on one side of zero" gating. This is a distributional refinement of L, not a new horizon.

Per project policy: open-ended, user decides. Both tools committed and ready to run.
