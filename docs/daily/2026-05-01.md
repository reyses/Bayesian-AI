# 2026-05-01 — Multi-TF v2 standalone runs + composite directional system

## TL;DR

Ran the full v2 standalone_research pipeline at 6 base TFs (1m, 5m, 15m, 1h, 4h, 1D) against `DATA/ATLAS/FEATURES_5s_v2/`. Built two composite tools (`v2_composite_directional.py` for OLS-mean, `v2_composite_quantile.py` for distributional Q_0.25/Q_0.75 voting, `v2_composite_l_aggregator.py` for direct cross-TF aggregation of standalone Analysis L outputs).

**Key finding: L's signed-MFE OLS works on own-cadence (62–71% accuracy, +2–6% lift) but naive cross-TF voting at a common 5m cadence drops to 52–53% per voter** — horizon mismatch (each TF predicts its own future window: 1m→1h ahead, 1h→8h ahead). Composite combinators salvage some lift: unanimous 5-TF agreement → 56.5% on 33% of data (+6.2% lift); magnitude-weighted |w|>5 → 59.8% on 13% (+9.6%).

**Operational read: trade per-TF L gates, not the naive composite.** A real composite system needs all TFs refit against a single common-horizon target.

## What this tells us about distributional vs point prediction

User's framing earlier: "we have the same problem classical physics has when evaluating electron position — by the time it reacts the electron is somewhere else; it might be right all along to treat price as a probability field."

Operationally validated:
- Bar-to-bar **direction sign** → R² = 0.0003 (Analysis B). Coin flip.
- **Conditional mean of signed MFE** (Analysis L) → R² = 0.06–0.11, gates to 73% at 20% coverage. The same data, different question.

The distinction is **distributional thinking**: stop asking "where will the price be next bar" and start asking "what's the conditional distribution of the trajectory; is it concentrated enough to bet". L is already a partial answer (mean of distribution); quantile regression (`v2_composite_quantile.py`) would extend to spread/skew.

Note: the QM analogy is rhetorical — distributional finance is mainstream (every option pricer, GARCH model, quantile regressor already does this). No physics imports into code per project policy.

## BREAKTHROUGH: MA alignment composite (`v2_composite_ma_alignment.py`)

User's counter-proposal that flipped the script: **don't strip the autoregressive vwap_w/price_mean_w features — use them directly as price-vs-smoothed-price comparators**, not regression inputs.

### Result

| Approach | Accuracy | Coverage | Lift |
|---|---:|---:|---:|
| **MA align combined ≥15** | **70.6%** | **19%** | **+17.8%** |
| **MA align vwap ≥7** | **70.5%** | **20%** | **+17.6%** |
| Standalone L \|pred\|>20 | 70.4% | 45% | +10.6% |
| Single-horizon 5-voter \|w\|>5 | 62.6% | 26% | +9.8% |
| 5-voter L-aggregator \|w\|>5 | 59.8% | 13% | +9.6% |

MA alignment matches standalone L on accuracy at half the coverage **and 67% more lift over baseline** (+17.6% vs +10.6%). Crucially, MA alignment is a **deterministic rule** — no model fit, no train/test risk, walk-forward stable by construction.

### The rule

For each of 8 TFs (5s, 15s, 1m, 5m, 15m, 1h, 4h, 1D):
```
vote_TF = +1 if price > vwap_w_TF
         -1 if price < vwap_w_TF
          0 if within tolerance (0.5 ticks default)
```
Sum the votes (`alignment_score`, range -8 to +8). Decision:
- LONG if `alignment_score >= 7` (7-of-8 TFs say price is above its smoothed value)
- SHORT if `alignment_score <= -7`
- FLAT otherwise

### Per-TF standalone strength

Single-feature rule (just one TF's vwap comparison):

| TF | VWAP acc | Lift | Verdict |
|---|---:|---:|---|
| 5s | 53.5% | +0.6% | window too short |
| 15s | 53.7% | +0.8% | window too short |
| 1m | 55.8% | +2.9% | weak |
| 5m | 59.0% | +6.2% | useful |
| **15m** | **61.7%** | **+8.9%** | strongest |
| **1h** | **61.6%** | **+8.7%** | strongest |
| 4h | 55.2% | +2.3% | too coarse for 5m decisions |
| 1D | 54.2% | +1.4% | too coarse |

The signal lives in the **15m and 1h smoothing windows**. 5s-15s smoothing windows don't capture meaningful trend; 4h-1D are too coarse for a 5m decision cadence.

### Why this beats the fitted composites

1. **Autoregressive features used as direct signals carry information that's lost when used as regression inputs.** The vwap_w trivially correlates with current price (r=0.99) — useless as an input to OLS predicting price level. But "price - vwap_w" carries direction signal cleanly because the 12-bar (or 5-bar) smoothing lag means current price has moved beyond where the smoothed average sits.
2. **No fitting = no overfit.** The 70.5% accuracy is computed on the last 20% of the timeline against a fixed deterministic rule. Robust to regime shift in a way OLS coefficients aren't.
3. **The user's "probability field" intuition validated.** Distributional thinking → "where is the conditional distribution of the trajectory concentrated?" MA alignment is a discrete approximation: count the TFs whose smoothed-price comparator says "above" or "below". When most agree, the conditional distribution is unambiguously on one side of zero. Same idea as quantile gating, computed without any model.
4. **Single feature handles ~70% of what L gets with 185.** 15m_vwap alone gives 61.7% direction accuracy. L on 5m base with all 185 features gives 62.3% global accuracy (with confidence-gating up to 70.4%). The vwap feature carries most of L's signal.

### Operational implication

**Ship this.** A NinjaTrader rule that compares current price to 8 TFs' rolling vwap (12-bar window for most, 5-bar for 1D, 9-bar for 5s/5m, etc. per `core_v2.N_BASE`) and trades when 7+ agree on direction.

Estimated NT8 implementation: ~20 lines in ZigzagRunner_v1.x with `LinReg(...)` indicator for the rolling vwap. No model file to load, no Python bridge needed for the directional signal itself.

## Regime connection: MA alignment IS a regime classifier

User pulled in prior regime research: [docs/memory/feedback_chop_edge_regime_filter.md](docs/memory/feedback_chop_edge_regime_filter.md) (2026-04-27) and [tools/atlas_regime_labeler.py](tools/atlas_regime_labeler.py) (2026-04-29 outputs in `DATA/ATLAS/regime_labels.csv`). The regime work and today's MA alignment are the same problem from two angles:

| Regime axis | Today's signal | Prior signal |
|---|---|---|
| Trend strength (UP/DOWN) | MA alignment ≥ 7 → 70.5% acc trade in alignment direction | atlas_regime_labeler labels these as UP/DOWN days |
| Chop / no-direction | MA alignment < 5 → no signal, FLAT | atlas_regime_labeler labels CHOP / TRANSITIONAL |
| Strategy fit | Trend-following on high-alignment | Counter-trend zigzag wins on chop, loses on trend (chop_edge_regime_filter d_OOS=0.77) |

**Reframed:** the MA alignment composite is an embedded trend-regime classifier. When alignment_score ≥ 7, you're in a regime where trend-following pays. When alignment is mixed, you're in chop where zigzag's bleed_score filter kicks in.

The right joint system isn't "one strategy with a confidence gate" — it's **regime-conditional strategy selection**:
- HIGH alignment → trade MA-align direction (trend follow, today's rule)
- LOW alignment + low bleed_score → trade zigzag counter-trend (prior finding)
- TRANSITIONAL / mixed → skip
