## Root cause: SFE cache bug

`LiveFeatureEngine._compute_features` cache keyed on `valid_idx` (bar count
up to current timestamp). After the bar store hit the 5000-bar trim limit,
every new bar got `valid_idx=5000` (newest always at the end after trim).
The cache returned stale SFE state from the first bar that reached 5000.

**This has been broken since mid-February.** Every live session since then
traded on stale features. The SFE cache fix:

```python
# BEFORE: cache key = valid_idx (collides after trim)
if cached and cached[0] == valid_idx:
    return cached

# AFTER: cache key = (valid_idx, latest_bar_ts)
cache_key = (valid_idx, float(sfe_input['timestamp'].iloc[-1]))
if cached and cached[0] == cache_key:
    return cached
```

## MockBridge (live engine replay harness)

New `live/mock_bridge.py` — drop-in replacement for NT8Client that replays
ATLAS bars. Enables offline parity testing without NT8:

```bash
python -m live.engine_v2 --mock --mock-day 2026_04_16 --headless
```

Key tweaks for mock mode:
- Step 0 cleans live folders on every startup
- `exclude_day` for LFE warmup (skip the replay day AND later days)
- Inbound queue maxsize=2 (backpressure vs 50K flood)
- Skip watchdog (time is compressed)
- Priority buffer for order ACK/FILL (drains before next bar)
- Save interval 500 bars (vs 3 in live — avoids 2K parquet writes)
