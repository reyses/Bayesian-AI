### Post-hoc vs engine-integrated mismatch — root cause

Post-hoc `apply_pivot_nn_filter.py` on `rm_oos.pkl` showed +$478/day. Engine-integrated showed −$226/day. Gap was the **FLIP regime illusion**: post-hoc just negates PnL sign, but asymmetric TP/SL is direction-specific (flipped SHORT has TP at entry−$50 / SL at entry+$10 — hitting the flipped SL is ~symmetric with the original SL, not mirror image).

Engine-integrated + no-FLIP confirmed this: FLIP bucket had 15% count-WR at −$5.46 mean. Post-hoc symmetry assumption was naive.

### Tier NN training failed

Trained `train_tier_direction_nn.py` on 5,268 tier trades (`iso_is.pkl`). Result: val AUC 0.52, test AUC 0.50. Model just predicts majority class (58-60% win rate). Diagnosis: base rate is high (tier rules do most filtering) + sample too small + insufficient separation.

Observation: tiers already extract most edge via rules; NN can't find more signal in post-rule residual.

User note: more labels would help. `blended_is.pkl` has 88k trades — 17× more. Script accepts `--trades <path>` so user can retrain on that.

### CNN has Long bias (not data problem)

After filtering: 91-97% of trades are LONG despite balanced training data (46% / 54% across iso trades, 52% / 48% in blended). CNN learned direction-specific feature patterns (probably regime drift in 2025 favored longs). Proposed fixes: stratified class weights, direction as input feature, separate long/short models. **Not implemented yet.**
