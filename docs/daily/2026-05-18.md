### TF rebuild pipeline (tools/sourcing/rebuild_nt8_from_1s.py)

Built three-mode tool (validate / write-new / overwrite). Aligns each
target TF to NT8 convention (`ts % period == period - 1`):

| TF | period | alignment | NT8-existing match |
|----|---|---|---|
| 5s | 5 | ts%5==4 | ✓ byte-identical |
| 15s | 15 | ts%15==14 | ✓ byte-identical |
| 30s | 30 | ts%30==29 | ✓ byte-identical |
| 1m | 60 | ts%60==59 | 7 days OHLC mismatch ($1-59 diff) |
| 5m | 300 | ts%300==299 | ✓ byte-identical |
| 15m | 900 | ts%900==899 | ✓ byte-identical |
| 30m | 1800 | ts%1800==1799 | ✓ byte-identical |
| 1h | 3600 | ts%3600==3599 | DIFFERS (existing uses 3540/2640) |
| 4h | 14400 | ts%14400==14399 | ✓ byte-identical |
| 1D | 86400 | ts%86400==86399 | DIFFERS (existing uses 75599=session-end) |

**Verification on 2026_05_15**: volume parity across all 10 TFs
(2,352,585 total volume — identical from 1s to 1D). Bar counts match
expected (10080 5s = 14h × 60min × 12; 1m=839; etc.).

### Data quality findings

**1m mismatches** (7 days, $1.50-$59.25 OHLC diff): existing 1m files
predate or use different export window. The 1s data is authoritative
(higher resolution); rebin produced correct values. Files NOT
overwritten — flagged in `nt8_rebin_validation.txt`.

**1h/1D offsets**: existing files use session-relative offsets (1h:
3540 mostly, 2640 on some days; 1D: 75599 = 16:59:59 ET cash close).
My rebuild uses standard period-1 offset. Both are valid; downstream
analysis tools should be alignment-agnostic.

**5s/15s/30s/5m/15m/30m/4h**: 100% match. The 1s-based rebin is
THE authoritative source for those TFs.
