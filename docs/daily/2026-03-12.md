# 2026-03-12 — TF Binning, OOS Compressed Mode, Phase 7 Live Replay

## Session Focus
Major architectural changes: TF-bucketed clustering, OOS/live parity fix,
Phase 7 live replay validation integrated into training pipeline.

## Changes Made

### 1. TF-Bucketed Clustering (`core/fractal_clustering.py`)
- Patterns now binned by timeframe BEFORE K-Means clustering
- Each TF bucket gets independent K-Means with its own target_k
- Prevents scale-mixing (15m swing templates no longer contain 1m scalps)
- Result: 403 templates from 13 TF buckets (vs previous 103 mixed)
- Bug fix: `valid_patterns` → `all_patterns` for transition matrix after refactor

### 2. OOS Compressed Per-Bar Mode (`training/trainer.py`)
- OOS now uses the SAME compressed per-bar path as live trading
- No discovery cascade in OOS — MarketState features only (16D, no ancestry)
- gate1_dist set to 9.5 (matches live aggression scaling)
- IS keeps full recursive discovery for library training
- Rationale: OOS should validate what live actually sees, not idealized discovery

### 3. Phase 7: Live Replay Validation
- New final phase in training pipeline
- Calls `live.launcher.main()` with `--replay-only` flag
- Uses ACTUAL live stack: LiveEngine → checkpoints → TBN → EE → HistoryReplayEngine
- Writes parity report: `reports/live/parity_report_*.txt`
  - Side-by-side OOS vs Replay: trades, WR, PnL, avg trade, trades/day
  - Exit reason breakdown, direction breakdown, gate funnel
  - Parity score + pass/fail verdict
- Saves warmed `live_brain.pkl` for live handoff

### 4. Live Launcher Cleanup
- Normal live (`python -m live.launcher`) now connects straight to NT8
- No more replay warmup in normal live mode — brain already warmed by Phase 7
- `--skip-replay` flag removed (no longer needed)
- `--replay-only` flag added: run replay, write parity report, exit
- `skip_replay` removed from LiveConfig

### 5. Pipeline Simplification
- `--fresh` alone runs full pipeline (Phases 1-7) — `--forward-pass` redundant
- Updated CHEATSHEET.md to reflect this

## Architecture Decisions
- **IS = full discovery** (14 TFs, parent chains, ancestry features) → trains library

- **Phase 7 = actual live stack** → integrity test of full production path
- **Normal live = no replay** → loads pre-warmed brain, connects straight to NT8

## Training Results (V7.0.0 — TF binning + OOS compressed + Phase 7)

### IS (12 months, full discovery)
- 3,298 trades, 96.6% WR, $39,736 PnL, $12.05/trade
- vs baseline (2026-03-09): $83,821 / 7,773 trades / 97.9% WR / $10.78/trade
- **~47% of baseline PnL** — TF binning produces more specific templates, fewer matches

### OOS (3 months, compressed per-bar)
- 643 trades, 100% WR, $8,200 PnL, $12.75/trade (all depth 8)
- vs baseline: $21,378 / 1,881 trades / 98.5% WR / $11.37/trade
- Oracle metadata blank — compressed path has no discovery, oracle can't classify
- All trades reported as "traded noise" despite being profitable (reporting artifact)

### Phase 7: Live Replay (5 days, actual live stack)
- 120 trades, 91.7% WR, $88.50 PnL, $0.74/trade
- **PARITY FAILED** (score 0.40)
- Exit breakdown: 82 stop_loss (-$0.25 avg), 28 envelope_decay (+$8.82), 9 peak_giveback (-$18.33)
- Direction: 116 SHORT / 4 LONG — heavy short bias
- Gate funnel: 264 skip_conviction (43%), 158 skip_headroom (26%), 69 skip_momentum (11%)
- Direction source: 98.3% live_bias — brain direction barely contributes

### Key Observations
1. TF binning cut IS PnL by ~53% — more specific templates = fewer but tighter matches
2. OOS compressed path gives artificially clean results (100% WR) because it bypasses live gates
3. Live replay is the honest picture: $0.74/trade with heavy SL churn
4. Peak giveback is broken in replay: 11.1% WR, -$18.33/trade (exit tuning doesn't transfer?)
5. Direction skew in replay: system defaults to SHORT (live_bias dominates)
6. The gap between OOS and Replay suggests OOS compressed mode is too generous

## Issues Found
- OOS oracle reporting: all trades classified as "noise" in compressed mode (no discovery = no oracle events)
- Parity failure: live stack sees the world very differently from compressed OOS
- Peak giveback exit tuning may not be loading/applying correctly in replay
- Direction: live_bias at 98.3% means brain direction corrections aren't being used

## Session 2: BarProcessor + OOS3 Architecture (evening)

### Architecture Overhaul
- **BarProcessor** (`core/bar_processor.py`): Shared per-bar decision loop replacing duplicated logic
  in trainer OOS, history_replay, and live_engine
- **One engine, one path**: OOS, replay, and live all use identical modules.
  Only difference is data source (parquet vs NT8) and side effects (orders, GUI)

### Parity Root Causes Found
1. `mode='live'` in EE activated `live_bias`/`live_momentum` directions → 98.3% SHORT bias
2. Different feature extraction (`_live_features` vs `_compressed_features`)
3. TBN warmup gap: replay starts cold, OOS has 48 days of belief accumulation

### Fixes Applied
- `live_engine.py`: `mode='live'` → `mode='oos'` for parity
- `live_engine.py`: Deleted `_live_features()`, uses `BarProcessor._build_features()` instead
- `live_engine.py`: Added `_init_bar_processor()` — `_check_entry()` delegates to BarProcessor
- `history_replay.py`: Migrated to use BarProcessor

### Pipeline Now: 6 Validation Gates
| Gate | What | Engine |
|------|------|--------|
| Phase 1-3 | Discovery + Clustering + Optimization | SFE |
| Phase 4 (IS) | In-sample forward pass | EE + ExitEngine (inline) |
| Phase 5 (OOS1) | Blind OOS | Same EE (inline) |
| Phase 6 | Strategy selection | — |
| OOS2 | Re-run with tier preference | Same EE (inline) |
| **OOS3** | **Bar-by-bar via BarProcessor** | **Same EE (BarProcessor)** |
| Live Sim | Paper trading via NT8 | Same BarProcessor |
| Live Real | Real money via NT8 | Same BarProcessor |

### Key Decision: OOS3 replaces Phase 7
- Old Phase 7: called `live.launcher --replay-only` (separate process, cold TBN)
- New OOS3: runs inside `run_forward_pass()` with `live_validation_days=999`
  - Brain/TBN fully warmed from OOS2
  - ALL days processed bar-by-bar through BarProcessor (mimics NT8 data feed)
  - Parity report compares OOS2 inline vs OOS3 BarProcessor
  - When OOS3 matches OOS2 → green light for live

### Files Changed
- `training/trainer.py` — OOS3 replaces Phase 7 (both `--forward-pass` and `--fresh` paths)
  - `live_validation_days` param on `run_forward_pass()`
  - `_fp_summary` override when all days go through BarProcessor
  - `_write_live_validation_report()` for parity comparison
- `core/bar_processor.py` — shared per-bar decision loop (created earlier this session)
- `live/live_engine.py` — mode='oos', BarProcessor integration, deleted _live_features()
