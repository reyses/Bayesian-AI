## The deployable rule

Two forward-available features computed from prior daily bars at session open:

```
prior_range       = yesterday's high - yesterday's low
range_compression = prior_range / mean(range over last 20 days)

bleed_score = (prior_range - 385.32) / 219.83
            + (range_compression - 1.0315) / 0.5502

if bleed_score > -0.34:   # = top 50% by IS
    SKIP today, no new entries
else:
    trade normally (counter-trend, all v1.4-RC machinery)
```

IS-calibrated constants are from the 48-day IS half (1/2-3/1/2026). Recompute
quarterly.

All four walk-forward / alternative-classifier tests say the **simple 2-feature
linear rule is the winner**. Resist the urge to add complexity.

`prior_range` and `range_compression` make the shortlist with min|d|≥0.41 in
ALL three splits. Signal is robust to where you cut IS/OOS.

### Convergent conclusion

The 2-feature linear bleed-score rule (`prior_range` + `range_compression`,
threshold z=-0.34) is the right MVP. **Do not** add:
- More features (overfits)
- Decision trees (overfits)

The simplicity is the strength. Ship it as-is.

## CRITICAL ROBUSTNESS FINDING (added 01:32 AM, kept for redundancy)

3-fold walk-forward (split 95 days into thirds, train on 2 / test on 1):

| Test fold | Period | Unfilt | Filtered | Lift |
|---|---|---|---|---|
| T3 | 3/18-4/24 | +$331 | +$2,119 | **+$1,788** ✓ |
| T2 | 2/08-3/17 | −$2,306 | +$1,813 | **+$4,119** ✓ |
| **T1** | **1/02-2/06** | **+$1,424** | **−$667** | **−$2,091** ✗ |

The filter **hurts performance during pure-chop regimes** (T1 had no bleed
days to catch; filter skipped winning days because prior_range was big due
to holiday volatility).

**Net 3-fold expected: +$1,272/test-period** (positive with variance).

**This means**: the filter is a regime-volatility play, not a universal alpha.
It works by catching bleed regimes when they occur. In pure-chop regimes
(rare in mature markets), it produces ~$60/day of false-alarm skips.

**Action item**: monitor live for ≥30 days after deploy. If 30-day filter-on
PnL < unfiltered would have been by >$1,500, manually disable. Document in
spec §8.5.

## Honest caveats

1. **All numbers are on a single 95-day window.** That window straddles
   one regime change (2/26). The rule may not generalize to a new
   regime change. Need a hold-out 30-day window before live promotion.
2. **Strategy underperforms passive long.** Buy-and-hold MNQ over the
   same window: +$247/day. v1.5-RC: +$50-80/day estimated. The strategy
   provides DIVERSIFICATION (uncorrelated with passive long), not max return.
3. **Multi-TF features overfit IS** when more than 2 are included. The
   2-feature rule is the cleanest walk-forward. Resist the temptation to
   add features.
4. **z=0.0 is empirically suboptimal.** Use −0.34 (MVP) or −0.5 (conservative)
   or +0.75 (aggressive) — not zero.
