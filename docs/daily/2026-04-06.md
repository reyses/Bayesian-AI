## What Changed
1. **5s features with live 1h/1D** — build_dataset_v2 loads per-TF from ATLAS, cross-day history. 1D_z_se now ALIVE (was zero all year). Daily context enables with-trend vs against-trend distinction.
2. **NMP gated to 1m boundaries** — 5s data captured for paths/EDA but entry/exit decisions only at minute marks. Prevents micro-noise trading.
3. **Extended regret window** — 120 bars lookback (10 min), 360 bars lookahead (30 min) at 5s. Captures full inverse peaks.
4. **AI entry from tree labels** — removed hardcoded z_se > 2.0 + vr < 1.0 gate. Tree classification IS the entry signal. Each of 128 branches is an ExNMP with its own physics.
5. **Z-score fallback** — higher TFs copy z_se/hurst/reversion_prob from lower TF when insufficient bars.

## Key Findings
- **1D features were dead all year** — build_dataset created fresh aggregator per day. 1D got 1 bar. Fixed: per-TF from ATLAS with cumulative history.
- **5s as trading resolution = disaster** — -$17/day. Too much noise. 5s as sensor array + 1m decisions = $484/day. Resolution for trading vs resolution for sensing are different.
- **Regret is a better AI than the AI** — it finds optimal physics per trade because it sees the outcome. Corrected trades carry that knowledge to the tree.
- **Epoch learning subtracts value** — H0 ($484) ≈ H1 ($488). Bayesian updates adjust exit bars that barely matter. No strategy competition, no competing strategy selection.
- **$2.30/trade is the real edge** — tiny per trade, compounds across 209 trades/day. Not fantasy numbers.

## Architecture Insights
- **ExNMP = tree branch** — each of 128 branches is a specialized strategy that works in its specific 79D context. The kill shot question: which have highest WR and why?
- **Sensor array model** — 5s/15s features update sub-minute (fast sensor), 1m makes decisions (main sensor), 15m-1h set bias (slow sensor), 1D sets daily context.
- **Physics over ML** — the corrected trades found contexts where specific directions work based on real price physics. The tree groups them. No black box.

## What Epochs SHOULD Learn (Next Session)
The current epoch phase is useless (shifts exit bars nobody reads). A real learn phase would:
1. **Per event: run ALL competing strategies** — which one wins at this 79D state?
2. **Strategy selector** — given competing strategies, pick the best based on accumulated evidence
3. **Confidence horizon** — each strategy has a decay curve. "Reversion confident for 5 bars, momentum for 20." Learn the horizon per strategy.
4. **CNN horizon insight** — "how many bars are you sure?" This is the hold duration confidence, not just the direction prediction.
