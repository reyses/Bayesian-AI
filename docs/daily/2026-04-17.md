# 2026-04-17 — Honest pipeline baseline, peak physics disproved, all tiers coin flips

## Morning: lookahead discovery in build_dataset

Yesterday's $900 peak live session implied the live features were now matching
training. But the *baseline* training $/day numbers looked too clean — so I
audited `training/build_dataset.py` for lookahead bias in the higher-TF
aggregation.

**Found it.** The inner loop used:
```python
idx = np.searchsorted(tf_ts, target_ts, side='right') - 1
```
That picks the TF bar whose start is at or before `target_ts`, but its OHLCV
contains 5s bars **forward** of `target_ts` up to the bar's end. So every
label at time T had future data baked into the higher-TF rows (≥1m).

Fix: shift the lookup back by the TF period so only *completed* bars are used:
```python
idx = np.searchsorted(tf_ts, target_ts - period, 'right') - 1
```

Moved output folders inside `DATA/ATLAS/` for cleaner structure:
- `DATA/ATLAS/FEATURES_5s/` (training)
- `DATA/ATLAS_OOS/FEATURES_5s/` (OOS)

Regenerated both. Took ~2 hours.

## KILL_SHOT peak physics analysis — no physics exists

User asked: can we build an exit that detects the peak *before* the giveback?
Built `tools/killshot_peak_physics.py` — reconstructs each trade's 5s path,
finds the peak bar, compares features AT peak vs ±3 bars (±15s).

**Every candidate physics signal failed:**

| Signal | Fire rate at peak |
|---|---|
| 1m velocity flips sign against trade | 3.3% |
| 1m acceleration flips against trade | 0.2% |
| 1m wick_ratio > 0.3 at peak (jump >30%) | 6.8% |
| Volume fades >20% across peak | 11.8% |
| 1m p_at_center > 0.6 at peak | 46.3% (but high before entry too) |

Largest Cohen-d across the peak was **0.19** (1m_wick_ratio). Everything
else <0.1. The peak is a statistical maximum over noise, not a detectable
feature event.

Back-test of 9 exit rules on the peak>$3 cohort (n=2,691):

| Rule | $/trade |
|---|---|
| **Baseline (natural exit)** | **+$11.61** |
| Fixed $10 target | +$11.22 |
| 1m accel negative | +$9.88 |
| Fixed $7 target | +$10.34 |
| p_center > 0.5 | +$7.69 |
| Velocity flip | +$6.83 |
| **50% trail from peak** | **+$3.40** |

The "common-sense" 50% trail was the *worst* rule. Every physics-based exit
loses to the natural exit. The existing kill_shot_center exit is already
~equivalent to a $10 fixed target in aggregate.

Verdict: **peak-physics exits are a dead end for KILL_SHOT.** The -$604
baseline on all 4,411 trades isn't an exit problem — it's an entry problem.
The ~1,720 trades with peak ≤$3 are structural losers that should never have
been taken.

## Regret on isolated tiers — all coin flips

Pivoted to the nn_v2 playbook: run regret on each isolated tier's trades and
see which have real directional edge. Built `tools/regret_on_isolated.py`.

Full results in `reports/findings/2026-04-17_iso_regret.md`. Headline:

| Tier | % counter | Counter WR | Flip@exit $/day |
|---|---|---|---|
| RIDE_AGAINST | 49% | 41% | +$850 |
| FADE_CALM | 49% | 38% | +$939 |
| MTF_BREAKOUT | 50% | 39% | +$115 |
| KILL_SHOT | 50% | 45% | +$84 |
| FADE_AGAINST | 46% | 38% | +$190 |
| CASCADE | 52% | 39% | +$49 |
| MTF_EXHAUSTION | 48% | 36% | +$23 |
| FREIGHT_TRAIN | 50% | 35% | +$382 |
| **Total** | **~49%** | **~40%** | **+$2,183** |

**Every tier sits at ~50% counter-flip rate.** Regret labels are meaningful
(counter-labeled trades have 35-45% WR — genuinely bad trades), but the
*boundary* between SAME and COUNTER is near-random. nn_v2 worked on NMP
because counter-rate was 30-35%; there was real direction to learn. At 50%,
a CNN has to learn a near-random boundary in 91D space.

**Oracle flip-at-exit upper bound: +$2,183/day across all 8 tiers combined.**
Even at 65% CNN flip accuracy (nn_v2 got 70.6%), realistic capture is
~$900-$1,300/day — but that assumes separability exists.

### Data caveat

`training/regret.py` has `LOOKAHEAD = 360 bars` commented "30 min at 5s" but
loads 1m price data — so the oracle window is actually 6 hours. 99% of
labels are `same_extended` or `counter_extended` because every trade's
"optimal" is "hold 6 hours and catch the swing." The `flip_at_exit` number
is still clean (no peak-chasing), but the absolute magnitudes of optimal and
flip@peak are inflated noise. Should cap LOOKAHEAD to something realistic
(15-30 min) before trusting per-tier optimals.
