# 2026-03-27 — 29D MTF Pipeline + Three-Layer CNN + Oscillation Proposal

## Summary
Built 29D feature pipeline (Phase A). Trained 29D StatePredictor (Phase B) — direction accuracy dropped to 56% vs 72% on 13D. Built L2 (DurationPredictor) and L3 (RetreatPredictor). OOS: $166/day on 29D (vs $1,609/day 13D baseline). IS three-layer: $602/day. Pivoted to oscillation-aware proposal after recognizing fixed exits don't transfer and models need market structure grounding.

## Key Findings
- **29D pipeline works**: zero-lookahead alignment across 1s/5m/15m/1h, all features normalized, 29.5s to build
- **29D model underperforms 13D**: corr 0.09 vs 0.24, dir acc 56% vs 72%. Extra MTF features hurt direction prediction — model struggles with 29D input
- **Walk-forward PnL is a bad metric**: 13D had -$11/day WF but $1,609/day OOS. 29D had -$4/day WF but $166/day OOS. Correlation and direction accuracy predict OOS performance, not WF PnL
- **Normalized features break the pipeline**: z-score normalized inputs with raw labels made training harder. Switching to raw 29D features improved results
- **L2 duration doesn't filter**: 89.6% of signals have MFE >= 4t, so L2 takes everything. Not a useful gate
- **IS three-layer**: $602/day, 75 trades/day, avg hold 13.9 bars, but 21K SL hits at -$225K
- **Duration exits are gold**: $452K from 7,148 trades ($63/trade avg). The signal is there
- **Natural 1m oscillation ~8 min**: user knowledge — 30-bar holds cross 4+ oscillation cycles, model can't distinguish normal dips from reversals
- **Fixed exits don't transfer**: IS-derived thresholds (BE, trail, SL) overfit. Need learned exits

## Architecture Pivot
Old approach: L1 direction → L2 duration → L3 retreat (fix exits with more models)
New approach: Measure market structure first, then build CNNs that answer specific questions:
1. What does trend physics look like? (1h/15m)
2. What direction is that trend? (15m/5m alignment)
3. What are the natural oscillations? (1m within trend, ~8 min period)
4. Optimal entry? (oscillation trough aligned with trend)
5. Optimal exit? (oscillation peak or trend ending)

Phase 1 is pure research (E1-E5 modules). Phase 2 CNN layers only if research validates.

## What to Look For
- E1: trend duration distributions per TF — do trends have measurable, predictable durations?
- E2: oscillation period confirmation — is ~8 min consistent across regimes?
- E3: cross-TF alignment — does 1m+5m trough alignment predict better entries?
- E4: oscillation-timed entry edge — must beat random by > 4 ticks to justify architecture
