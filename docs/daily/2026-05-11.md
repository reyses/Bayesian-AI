# 2026-05-11 — v6 3-body breakthrough + regime-conditional finding

## Session arc

User pointed out (with a screenshot of a clean swing trade) that the 1h HL is
for **position context**, the 15m CRM is the **trade target**, and the 15s
CRM is the **trigger**. I had been firing on 1h-anchor extension instead of
15m-anchor extension — wrong timescale.

## v6 3-body architecture (`tools/sim_v6_3body.py`)

```
1h HL    → POSITION (slow context; where in the structural band; loose gate)
15m CRM  → TARGET (medium reversion target; trades aim to revert TO 15m mean)
15s CRM  → TRIGGER (fast cusp signal; slope sign-flip)

SHORT cusp:
  z_15m_crm  >= +1.5       (extension above 15m mean → room to revert)
  slope_15s flipped + → -  (cusp at top of 15s oscillation)
  pos_in_band <= 1.2       (1h context: not in extreme rally)

LONG cusp:
  z_15m_crm  <= -1.5       (extension below 15m mean)
  slope_15s flipped - → +  (cusp at bottom)
  pos_in_band >= -0.2

Exits:
  TARGET: z_15m_crm crosses back through 0  (revert to 15m mean)
  STOP:   opposite extension (e.g. SHORT stops if z_15m >= +2.7)
  TIME:   45 min
```
