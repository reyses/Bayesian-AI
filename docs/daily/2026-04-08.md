### SFE Pipeline Audit
- **CRITICAL bug found**: OHLCV window mismatch — SFE used tail(300) but features used FULL history
  - variance_ratio, vol_rel, bar_range computed on different window than z_score
  - Fixed: created `nn_v2/compute_79d.py` as single source of truth
  - Both live_engine.py and maintenance.py now call `compute_79d_from_aggregator()`
- **Dashboard regression center bug**: `getattr(dict, 'regression_center')` always returned default
  - Fixed: `states_by_tf['1m']['state']` unwrap before getattr

### Dashboard Cleanup
- Removed old tick-based SE bands + rolling mean center (was drawing jagged lines from 1s data)
- Removed purple 5s tick line
- Chart now uses only 1m data: blue=1m close, white dashed=1m regression center
- z_se signal meter moved from TICK_UPDATE to BAR_1M handler (updates once per minute)
- Chart scaling uses 1m + center data range

### Live Engine Bugs Fixed (from real trading)
1. `record_trade(pnl, trade.get('peak', 0))` — passed float instead of trade dict
2. Unicode `->` in log messages — cp1252 encoding error on Windows
3. `end_trade` -> `finish_trade` — wrong method name on TradeLogger
4. Extra `pnl` arg in finish_trade call — signature mismatch
5. Dashboard buttons (BUY/SELL/FLATTEN) not wired — `shared_state['manual_order']` never read
6. Orphan position detection — NT8 has position but engine thinks flat after crash
7. Stale order cleanup — periodic pruning of stuck PENDING/WORKING orders
8. Window close sequence — removed duplicate flatten, let _shutdown() handle everything

### Manual Trades via Dashboard
- BUY/SELL now go through BlendedEngine (`inject_manual_trade()`) for full CNN exit management
- FLATTEN closes through both BlendedEngine and NT8
- MANUAL tier treated same as BASE_NMP for exits (CNN hold + CNN risk + hard stop)

### Giveback Stop (DISABLED)
- Added but disabled (GIVEBACK_MIN_PEAK=99999) after discussion
- Judging system on single trade is reactive/overfitting
- CNN hold bias toward HOLD is known but needs more data to evaluate
