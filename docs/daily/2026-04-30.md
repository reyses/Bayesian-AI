### Findings — quantitative (aggregate Cohen's d)

Across 1,912 fade-tier trades (BASE_NMP + FADE_CALM + FADE_AGAINST):

| Feature | Cohen's d (winner-loser) | Direction |
|---|---:|---|
| 1m_reversion_prob | **+0.472** | winners HIGHER |
| 1m_hurst | -0.255 | winners LOWER |
| 1m_variance_ratio | -0.219 | winners LOWER |
| 1D_dmi_diff | -0.141 | winners on DOWN-day |
| (rest near zero) | | |

z_se features have d ≈ 0 — they ARE the trigger, so the trigger doesn't
distinguish winners from losers within the triggered set. That's the
whole point: physics-CONDITIONAL-on-trigger is what matters.

### Findings — heuristic threshold filter (NEGATIVE result, important)

Hypothesis: skip fades with low revprob and/or high hurst → expect +PnL
delta similar to the LinReg slope filter (+$33k overnight).

Reality: every threshold combo REDUCES total PnL. The filter raises
per-trade quality (WR ↑, $/trade ↑) but drops trade count faster than
it raises win density. Skipped trades have lower per-trade PnL but
still net positive in aggregate.

Best heuristic combo (revprob>=0.85): only +$240 vs baseline. Not
exploitable as a threshold rule.

**Implication**: physics-at-entry contains real info but extracting it
needs a NONLINEAR decision (CNN). The 70% cnn_flip accuracy is the
right tool; thresholds aren't. This is a sharper validation of the
CNN thesis: not "the rule says skip these," but "you need a model that
can do nonlinear feature combinations."

**Counter-intuitive corollary**: the +$33k LinReg slope filter
discriminates on a DIFFERENT axis than these per-bar physics features
(none of them encode local slope structure directly). Two filters,
two different discrimination axes, both real.

### Known caveat
With a 1-day analysis window, 1D TF features are constant within the window so their correlations come back as 0 (or NaN→0). For meaningful macro-TF feature correlations, run with a multi-month window.
