## Session Summary

## Key Discovery: F_momentum Parity Break
- Live F_momentum median = 12.3 vs ATLAS median = 218.5 (17x difference)
- Root cause: PID controller uses `np.cumsum(z_scores)` — integral term accumulates over full history
- ATLAS has 417K contiguous 1m bars. Live had ~2.5K bars (resampled from 15s)
- cumsum starts from zero in live → near-zero F_momentum → sensor gate blocks EVERYTHING
- Evidence: 303,897 sensor blocks in 2 hours of live trading. OOS took 25 trades in same window (+$549)
- Fix: pre-compute states from ATLAS, load at live startup
