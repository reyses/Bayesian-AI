# 2026-04-24 — v2 pivot CNN dead end, zigzag-only sweep, NT8 strategy, LightGBM pivot

## Summary

CNN on v2 features collapsed to random (AUC 0.51 on 9,407 v2 engine trades). Diagnostic proved v2 features DO carry signal on v1's 299 curated trades (AUC 0.60), so the CNN architecture — not the features — is the block. Pivoted to:

1. Pure-rule zigzag backtest (deterministic, no ML)
2. NT8 Strategy deployment (ZigzagRunner v1.2 with CSV log + trailing stop)
3. LightGBM as the correct model for this tabular finance task (design doc drafted)

## What landed

### v2 CNN exhausted
- Temporal stack (T=12, Conv2d(T, ...)) gave AUC 0.516 vs static 0.512 — trajectory inputs did not unblock the CNN
- Root cause argued: CNN is wrong architecture for our tabular non-spatial 8×23 grid at 5-10k samples. TF ordering is arbitrary; no locality to exploit.

### Zigzag-only backtest (tools/zigzag_backtest.py)
Pure geometric rule: enter on R-retracement confirmation, reverse at next confirmation, EOD close.
- **Theoretical model** (entry at extreme±R): R=30 yields +$986/day IS, +$1,059/day OOS, 80%+ day WR
- **Realistic model** (1s-bar random fill + $2 round-trip commission): ALL R thresholds flip NEGATIVE. R=30 → -$132/day IS, -$224/day OOS.
- Conclusion: theoretical edge exists but 1m confirmation lag destroys it on market orders. Limit orders at extreme±R would be the recovery path.

### NT8 Strategy deployment
- `docs/NT8_ZigzagRunner.cs` (v1.2.0) → mirrored to `Documents/NinjaTrader 8/bin/Custom/Strategies/`
- Per-trade CSV logging via `OnExecutionUpdate` (handles reversal fills correctly)
- Trailing stop: arms at 10-pt profit (MNQ $20), trails 5 pts behind HWM (MNQ $10). Does not loosen. Resets on every entry.
- Post-cutoff exit: reverses exits the position even if no new entry allowed (prior version silently held)
- User will run Strategy Analyzer to compare against Python backtests

### LightGBM design doc (docs/JULES_PIVOT_LGBM_V2.md)
Comprehensive spec for replacing CNN with LightGBM on the same v2 feature set:
- 4 reasons CNN was wrong (no locality, arbitrary TF order, translation invariance N/A, data volume too small for DL)
- Success tiers: green ≥0.62 AUC, yellow 0.55-0.62, red <0.55
- Explicit OPEN DECISIONS flagged for counterbalance LLM review before implementation (label threshold, temporal deltas, CV scheme, feature selection, class weighting, per-direction models, ship criterion)
- Phase A (build+smoke) → Phase B (eval) → Phase C (diagnostics) → Phase D (engine integration)
