## Session 2 — Training/Live Parity Refactor

### SFE Review Finding
Reviewed how SFE is computed in training vs live. Found 3 mismatches:
1. **SFE window**: bulk build_dataset.py fed full day (~1440 bars), live uses tail(300)
2. **bars_held cadence**: training at 5s incremented 12x/min, live at 1m = 1x/min
3. **on_state cadence**: training calls engine every 5s, live only at 1m close

### Changes (18 files, commit `180fe7b5`)
1. **Renamed build_dataset_v2.py → build_dataset.py** — dropped _v2 from filename,
   output dirs (FEATURES_79D_5s_v2 → FEATURES_79D_5s), constant (SFE_TAIL → SFE_WINDOW).
   Bulk builder already archived. Sequential is now the only path.
2. **bars_held = timestamp-based** — `int((ts - entry_ts) // 60)` instead of `+= 1`.
   Cadence-independent: same value whether on_state called at 5s, 15s, or 1m.
   CNN hold/risk bars_norm now consistent between training and live.
3. **Live engine fires at 5s** — on_state triggers at 5s bar close (was 1m).
   Entries still gated by `is_1m` check inside BlendedEngine.
   Exits checked 12x/min (was 1x) — matches training responsiveness.
4. **inject_manual_trade classifies tier** — was hardcoded MANUAL (no exit logic).
   Now runs _classify_full_tier() to pick CASCADE/KILL_SHOT/FADE_CALM/etc.
   Manual trades get proper tier-specific exit management.
5. **maintenance.py bug fix** — was checking FEATURES_79D_5s for missing days but
   building --resolution 1m. Would always find "missing". Now checks FEATURES_79D_1m.
6. **Directory renames on disk** — FEATURES_79D_5s_v2 → FEATURES_79D_5s,
   FEATURES_79D_1m_v2 → FEATURES_79D_1m.
