## Peak signature — what defines "exit this trade"

For each of 452 TREND_FOLLOWER trades, found peak bar, compared entry→peak
feature vectors normalized by entry std dev:

| Feature | Entry | Peak | Exit | d/σ | Dimension |
|---|---:|---:|---:|---:|---|
| **1m_p_at_center** | 0.047 | 0.395 | 0.297 | **+10.32** | Position (at center) |
| 1m_variance_ratio | 1.323 | 0.795 | 0.995 | **-1.81** | Regime (chaos settled) |
| 1m_reversion_prob | 0.485 | 0.870 | 0.777 | **+1.16** | Probability (OU reversion done) |
| 15s_dmi_gap | 35.6 | 18.2 | 20.6 | -1.44 | — |
| 1m_wick_ratio | 0.253 | 0.477 | 0.374 | +1.35 | — |

Top 3 chosen because:
1. They cover three ORTHOGONAL physics dimensions (position / regime / probability).
2. `variance_ratio < 1.0` is the EXACT INVERSE of the entry gate `vr > 1.0` —
   perfect symmetry, free confirmation.
3. `p_at_center` dominates at +10σ — a one-feature signal would work, but
   three provides noise immunity (pass-through vs arrival).

## Exit rule shipped (TREND_FOLLOWER only, clean physics)

```python
Exit when ALL of:
  1m_p_at_center > 0.35     (price at regression mean)
  AND 1m_reversion_prob > 0.80   (OU says reversion done)
  AND 1m_variance_ratio < 1.0    (chaotic regime settled)
```

Symmetric with entry:
- Entry: `|z|>2 + |1m_vel|>10 + vr>1` (band edge, moving, chaotic)
- Exit: `p_center>0.35 + reversion>0.80 + vr<1` (at center, done, settled)

Exit reason label: `trend_follower_peak`. Inverse-signal exit (general fallback)
is retained for trades that pass WITHOUT all three peak signals firing.

## Methodology captured

Three-question process for tier building:
- Q1: Are entries right? → peak-bucket analysis (>80% peak>$20 = yes)
- Q2: What persistent signal says we're wrong? → bar-N path analysis
- Q3: What do all peaks have in common? → entry→peak feature delta by σ

See `memory/feedback_tier_three_questions.md` for the full method +
anti-patterns.
