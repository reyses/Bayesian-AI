## Session 2: Pipeline Phase Restructuring

### Problem Identified
Phase 5 (Strategy Selection) was fundamentally broken:
1. **Circular grading**: graded templates on IS data they were discovered from
2. **No real filtering**: tier map barely influenced the forward pass
3. **Wrong position in pipeline**: ran between IS and OOS instead of after

### Design Decisions
- **Playbook = final output, not an input** — never feeds back into training/IS/OOS
- **Tiers are a tiebreaker, not a gate** — when multiple candidates compete on the same bar,
  prefer proven (Tier 1) over toxic (Tier 4). No template is ever blocked.
- **Strategy grades on OOS** (blind validation), not IS (circular)
- **No iterations** — one IS pass, one OOS pass, then lock the playbook
- **OOS₂ verification** — re-run OOS with tiebreaker active, compare delta

### New Pipeline Flow
```
Phase 1: Discovery     — fractal pattern scan
Phase 2: Clustering    — template generation
Phase 3: Optimization  — fission + DOE tuning
Phase 4: IS Backtest   — in-sample replay, tune exits/gates/direction
Phase 5: OOS Validation — blind out-of-sample (no tier influence)
Phase 6: Strategy      — grade templates on OOS results, assign tiers
OOS₂ Verify            — re-run OOS with tier tiebreaker, measure delta
```

### Changes Made
- **Removed**: `--passes` flag (iterations overfit IS), `--min-tier` flag (hard gate)
- **Moved**: Strategy selection to after OOS, reads `oos_trade_log.csv` instead of brain history
- **Added**: `tier_preference` parameter — conditionally activates tier scoring in candidate selection
- **Added**: `_print_oos_comparison()` — OOS₁ vs OOS₂ delta report
- **Renumbered**: all phase labels 1-6 with descriptive names throughout codebase
- **Removed**: iteration loop, pass comparison report (dead weight)

### Analysis U Design (dp/dt psychohistory)
- **Concept**: Don't predict the path, predict the destination with a confidence interval
- Like psychohistory: individual bars unpredictable, but patterns with similar 192D
  characteristics (velocity, momentum, hurst, coherence) resolve to predictable ranges
- **Method**: k-NN in 192D feature space → neighbor signed MFE distribution → [P10-P90] CI
- **Key outputs**: CI coverage, direction from P50, ambition ratio, dp/dt signature,
  counter-trend context, CI quality by regime volatility
- Observation-only design: hook into pipeline without affecting decisions
