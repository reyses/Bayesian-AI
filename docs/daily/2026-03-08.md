## Session 2: Pipeline Phase Restructuring

### Problem Identified
Phase 5 (Strategy Selection) was fundamentally broken:
1. **Circular grading**: graded templates on IS data they were discovered from
2. **No real filtering**: tier map barely influenced the forward pass
3. **Wrong position in pipeline**: ran between IS and OOS instead of after

### Design Decisions
- **Playbook = final output, not an input** — never feeds back into training/IS/OOS
- **Tiers are a tiebreaker, not a gate** — when multiple candidates compete on the same bar,
  prefer proven (Tier 1) over toxic (Tier 4). No template is ever blocked.
- **Strategy grades on OOS** (blind validation), not IS (circular)
- **No iterations** — one IS pass, one OOS pass, then lock the playbook
- **OOS₂ verification** — re-run OOS with tiebreaker active, compare delta

### New Pipeline Flow
```
Phase 1: Discovery     — fractal pattern scan
Phase 2: Clustering    — template generation
Phase 3: Optimization  — fission + DOE tuning
Phase 4: IS Backtest   — in-sample replay, tune exits/gates/direction
Phase 5: OOS Validation — blind out-of-sample (no tier influence)
Phase 6: Strategy      — grade templates on OOS results, assign tiers
OOS₂ Verify            — re-run OOS with tier tiebreaker, measure delta
```

### Changes Made
- **Removed**: `--passes` flag (iterations overfit IS), `--min-tier` flag (hard gate)
- **Moved**: Strategy selection to after OOS, reads `oos_trade_log.csv` instead of brain history
- **Added**: `tier_preference` parameter — conditionally activates tier scoring in candidate selection
- **Added**: `_print_oos_comparison()` — OOS₁ vs OOS₂ delta report
- **Renumbered**: all phase labels 1-6 with descriptive names throughout codebase
- **Removed**: iteration loop, pass comparison report (dead weight)

## Session 6: Trajectory Extrapolation Insight (Brain Key Revolution)

### The Chain of Reasoning
1. Analysis E: dp/dt grouping gives 3x direction R² lift → clustering works
2. Analysis D: delta features (rate-of-change) give 5x direction R² lift → trajectory > snapshot
3. Analysis P: paired points (A,B) in 192D give 98.8% direction → transitions are deterministic
4. **Key insight**: P isn't leakage — it's interpolation/extrapolation in 192D space
5. **Octa-point extrapolation**: use 8 sequential 192D snapshots to extrapolate the 9th
   - 1 point = position (useless, R²=0.01)
   - 2 points = direction (98.8%, but needs future state)
   - 8 points = full trajectory: position, velocity, acceleration, jerk, curvature
   - Can FIT the curve through 192D space and PROJECT where it goes next

### This IS the Bayesian Brain
- The brain already maps state → outcome distribution via lookup table
- Current key: `(template_id, direction)` — a discrete hash
- **New key**: trajectory hash — discretize the 8-point path shape through 192D
- Brain has seen similar 8-point trajectories → knows where 9th landed historically
- Same data structure, same accumulation, same lookup — just better keys
- No new model needed, no k-NN at runtime, no regression coefficients to serialize

### Observation Layer Integration (approved)
- Observation-only hook: log predictions alongside trades, don't affect decisions
- At entry: compute trajectory hash from last 8 bars of 192D states
- Each bar: update trajectory, re-query brain for updated CI
- At exit: score prediction accuracy
- Promote to decision layer only after proving coverage on OOS
