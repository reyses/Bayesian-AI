## Session 3: OOS3 Incremental TBN (true live simulation)

### Problem
OOS3 called `prepare_day()` with the full day's 15s data (2400 bars), giving TBN workers
visibility of the ENTIRE day at bar 0. In live, NT8 sends history bars (past only), and
`prepare_day()` is refreshed every 240 bars (hourly) with only accumulated data.

Higher-TF workers (30m, 1h) had complete bars from future data — a subtle lookahead.

### Fix
- `batch_compute_states()` stays batch (backward-looking regression = identical to incremental)
- For OOS3 days, `prepare_day()` is **skipped** at the top of the day loop
- Inside the OOS3 bar loop, TBN is refreshed every 240 bars with only data up to current bar:
  - `df_15s[:current_bar]` — no future 15s bars
  - `_df_5s[timestamp <= current]` — no future 5s bars
  - `_df_1s[timestamp <= current]` — no future 1s bars
  - `_df_4h` unchanged (monthly ATLAS = always historical)
- Matches `live_engine.py` line 660: `if self._bar_i % 240 == 1: prepare_day()`

### Files Changed
- `training/trainer.py` — OOS3 incremental TBN refresh, `_is_live_val_day` guard

## Session 4: OOS3 Bug Fixes + Successful Run

### Bugs Found & Fixed
1. **Incremental TBN too slow**: Refreshing TBN every 240 bars recomputed states for all
   accumulated 1s/5s data (600k+ bars × 50 refreshes = massive GPU load). Hung the process.
   - Fix: Single `prepare_day()` call at start of OOS3 day (matches inline OOS).
   - Rationale: `batch_compute_states()` is backward-looking, so full-month data is safe.

2. **`_live_val_start_idx = -2`**: `len(files) - live_val_days = 3 - 5 = -2`, treating
   ALL files as OOS3 (brain never warmed, 0 trades).
   - Fix: `_n_val_files = min(1, len(files) - 1)` — only last file goes through OOS3.

3. **`UnboundLocalError: _out_dir`**: Set inside inline OOS loop, skipped when all files OOS3.
   - Fix: Default `_out_dir` before the loop.

4. **Sub-TF data capped to 0-1 bars**: Capping 5s/1s to `timestamp <= first_ts_of_day` with
   monthly files yielded near-empty TBN workers → 0 conviction → 0 trades.
   - Fix: Pass full month sub-TF data (backward-looking, safe).

5. **Bar index double-offset**: `bar_idx + rp` double-counted regression period. `bar_idx`
   already starts at `rp=21`, adding 21 shifted price by 5.25 min.
   - Fix: `_lv_row_idx = _lv_result['bar_idx']` (no offset).

6. **For loop inside except block** (root cause of 0 trades): The `for _lv_i, _lv_result in
   enumerate(_states_15s):` loop was accidentally indented inside the `except Exception:` block.
   When `prepare_day()` succeeded (no exception), the entire bar processing loop was skipped.
   - Fix: Dedented the for loop to be at the same level as try/except.

### Entry Improvements (DEEP_RESEARCH_ENTRY_IMPROVEMENTS.md — 5 items)

**A. Regime-Aware Gate 0** — `_classify_regime()` returns strong_trend/developing/exhausting/range/chop.
