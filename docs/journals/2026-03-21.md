### Architecture Decisions
1. **Only ONE cumulative field existed**: term_pid (cumsum of z_scores). Everything else
   (F_momentum, DMI, ADX, velocity, volume_delta) was already per-bar or rolling window.
2. **Sensor gate should be proportional**: absolute thresholds don't scale across regimes.
   Z-score relative to rolling stats auto-calibrates.
3. **Fake peak filter belongs on EXIT side, not ENTRY**: high vol+fm at entry = good entry.
   High vol+fm DURING trade = the move hasn't peaked yet, hold.
4. **The counterfactual engine is the goat brain**: every trade and skip spawns phantom
   trades with alternative parameters. Optimal settings emerge from real-time what-if.

## Session 2 — IS BarProcessor Refactor (Mar 21 afternoon)

### The Refactor
Deleted 1,029-line IS inline forward pass. Replaced with ~350-line BarProcessor-driven loop.

**What was deleted:**
- Inline peak detection + cat/crow hooks (duplicated BarProcessor logic)
- Inline exit evaluation (gathered TBN signals, called EE directly)
- Inline entry candidate building (compressed + multi-TF + peak)
- Inline equity gate, position opening, trade recording
- Ping-pong direction override logic

**What was kept (loop wrapper bookkeeping):**
- Calendar day boundary flush + progress bar + dashboard push
- PID analyzer tick (previous bar state)
- Oracle record assembly at entry + completion at exit
- Equity/drawdown tracking after each trade
- Trade replay capture (I-MR analysis)
- Decision matrix records + audit (TP/FP/FN classification)
- Daily DD stop ($500 buffer)
- Maintenance window flatten
- Ping-pong flip after exit

**How it works now:**
1. BarProcessor hooks: `pre_exit_eval` (1s sub-bar data), `modify_pnl` (slippage), `on_entry` (equity gate)
2. `_bp.process_bar()` handles TBN tick, cat/crow, peak detection, candidate building, EE entry/exit, trade recording
3. Loop reads `result.action.type` for entry bookkeeping, `result.trade_completed` for exit bookkeeping
4. `_on_trade_completed()` helper: oracle, equity, replay, dashboard markers

**Files changed:**
- `core/bar_processor.py` — added `trade_mfe_ticks` + exit signal fields to trade dicts (_handle_exit + force_close)
- `training/trainer.py` — deleted 1,029 lines, replaced with ~350 lines via process_bar()
- `live/live_engine.py` — added maintenance flatten (was missing: blocks entries but never force-closed)

**Bugs fixed by refactor:**
- IS peak detection used lookahead (state N+1 instead of N) — now uses current bar state via BarProcessor
- Crow phantoms will now fire in IS (was blocked because IS bypassed _build_candidates)
- IS and OOS now use identical code path for entry/exit decisions

**Risk assessment:**
- Oracle records may have slightly different field values (MFE computed in BP now vs inline before)
- Peak detection behavior changes (no more lookahead) — could affect trade count/timing
